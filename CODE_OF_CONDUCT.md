# Code of Conduct — `oqs-cbg-pipeline`

This project follows a light, harbour-style code of conduct.

## Operating principles

- **Be constructive.** Frame critique as a proposal for the next version.
- **Attribute generously.** Cite prior work, including informal contributions in issues, discussions, and reviews.
- **Refuse to flatten substantive disagreement into consensus.** Holding `UNDERDETERMINED` is a valid stance. Forcing premature `COMPATIBLE` or `INCONSISTENT` to maintain an appearance of progress is not.
- **Maps are not territory.** Repository outputs are coordinate-dependent under the Hayden–Sorce gauge. Do not reify computational artefacts as physical claims.
- **Respect the Sail discipline.** Parameter freezing, stewardship flags, coordinate-choice annotations, and DG cause labels exist to protect the citation scope of every contributor's work.

## What is not tolerated

- Bypassing the unidirectional consumption rule (the repository never modifies the Ledger).
- Removing or weakening stewardship flags without the procedure in `docs/stewardship_conflict.md` Rule 4.
- Silent post-hoc adjustment of frozen parameters; see `docs/benchmark_protocol.md` §4.3.
- Personal attacks, harassment, or discrimination.

## Enforcement

The steward (currently U. Warring) is responsible for enforcement at the repository layer. Issues at the Sail or Ledger layer route to local stewardship and Council deliberation respectively, per `docs/endorsement_marker.md`.

## Reporting

Issues may be raised via repository issue tracker or directly to the steward. Anonymous channels are not supported because the resolution process requires correspondence; pseudonymous reporting is acceptable.
