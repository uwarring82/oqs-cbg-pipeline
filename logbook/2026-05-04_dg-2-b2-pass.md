### Hermiticity and HPTA precondition gates

The runner enforces three gates per case in this order:
1. **Hermiticity-of-omega**: `||omega - omega^dagger||_F ≤ 1.0e-14`.
2. **HPTA**: `||sum_{i,j} omega_{ij} V_j^dagger V_i||_F ≤ 1.0e-14`.
3. **K comparison**: `||K_computed - K_expected||_F / max(||K_expected||_F, 1) ≤ 1.0e-10`.

Hermiticity is gated first because a non-Hermitian omega makes the HPTA residual ill-typed (it can carry an anti-Hermitian component) and the K comparison downstream meaningless. Both precondition gates short-circuit on violation with dedicated diagnostic notes; this is exercised synthetically in [`tests/test_benchmark_card.py::test_run_card_b2_hermiticity_gate_short_circuits_on_violation`](../tests/test_benchmark_card.py) (non-Hermitian fixture rejected with Hermiticity-specific note) and [`tests/test_benchmark_card.py::test_run_card_b2_hpta_gate_short_circuits_with_hermitian_omega_but_no_hpta`](../tests/test_benchmark_card.py) (Hermitian-but-not-HPTA fixture rejected with HPTA-specific note, confirming gate ordering).

For the three frozen fixtures:

- σ_z fixture (V_1=I, V_2=σ_z, β=0.5): `omega = [[1, iβ], [-iβ, -1]]` is Hermitian by construction (`omega_21 = (omega_12)*`); HPTA residual is `(1 - 1) I + (iβ - iβ) σ_z = 0` identically.
- σ_x fixture (V_1=σ_x, V_2=I, β=0.5): same omega structure (Hermiticity unchanged by V_i basis swap); HPTA residual collapses to `0` using `σ_x² = I`.
- diagonal-only fixture (V_1=I, V_2=σ_z, ω = diag(1, -1)): real-diagonal omega is trivially Hermitian; HPTA is `I - σ_z² = 0`.
