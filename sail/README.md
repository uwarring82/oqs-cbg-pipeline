## What a Sail is

In the Open-Science Harbour vocabulary, a *Sail* is an interpretive and programme-setting document. It motivates action, encodes discipline, and protects the underlying Ledger from misuse, but it makes no validation claims of its own. Sails do not require Council clearance; they operate under local stewardship.

This Sail anchors to the Council-cleared Ledger entry in [`../ledger/CL-2026-005_v0.4.md`](../ledger/CL-2026-005_v0.4.md) and scopes the numerical programme this repository implements.

## Authority

The Sail is the highest-authority document at the repository's own layer. Substantive changes to the Sail (anything beyond typo-level corrections) trigger a version bump (v0.5+) and a fresh round of stance review (Steward + Architect + Scout + Verifier composite). Substantive changes that bear on Ledger entries route through fresh Council deliberation, not through Sail revision.

## Reading order

Read the Sail before reading any code. The repository's structure, gates, and discipline derive from the Sail; without that context, the code structure will appear arbitrary.

## Intermediate versions

Earlier versions (v0.1, v0.2, v0.3) are not vendored. Their content is summarised in the Sail's revision log (§12 of v0.4). If a future steward needs the full text of an earlier version, it is recoverable from the project's history outside this repository.
